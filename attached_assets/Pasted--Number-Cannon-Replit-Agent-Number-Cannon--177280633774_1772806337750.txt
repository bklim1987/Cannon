# 数字大炮 (Number Cannon) — Replit Agent 开发提示词

## 项目概述

创建一个名为"数字大炮 (Number Cannon)"的双人对战教育数学游戏。这是一个 React 单页应用，将部署到独立 URL，并通过 iframe + postMessage 集成到外部锦标赛管理系统 (Tournament Hub)。

**目标用户**：10-11岁学生，在课堂大屏触摸屏上操作。

**核心玩法**：两名玩家分别在屏幕左右两侧操作各自的大炮，通过发射质数炮弹来击碎从上方下落的合数怪物。怪物被命中后显示剩余数值（可视化短除法），数值变为1时被消灭。

---

## 技术要求

- **框架**：React（使用 Vite 脚手架）
- **语言**：JavaScript（不用 TypeScript）
- **样式**：CSS Modules 或内联样式，不使用 Tailwind
- **部署**：Replit 内置部署
- **响应式**：优先适配横屏大屏（1920×1080 或类似），也需兼容平板
- **触屏优化**：所有交互按钮至少 48px 触控区域，使用 `touch-action: manipulation` 消除300ms延迟

---

## 游戏模式

本次只实现 **模式A（锦标赛简化版）**：

- 游戏时长：**60秒**
- 怪物持续生成并下落
- 时间到后结算分数
- 分数高者胜

5. **锁定提示**（条件显示）：射错时显示锁定倒计时

## 操控方式（已验证，非常重要）

### 移动大炮
- **点击网格上的任意一列**即可将大炮跳转到该列
- 不使用左右箭头按钮
- 整列的所有格子都可以点击来移动
- 大炮当前所在列用浅色高亮显示，底部用三角箭头标识

### 发射炮弹
- 点击底部的质数按钮发射
- 炮弹向正上方发射，命中当前列**最低位置**的怪物
- 如果当前列没有怪物，炮弹消失，不处罚
- **没有冷却时间**，玩家可以快速连续点击

### 触屏要求
- 所有按钮使用 `onPointerDown`（而非 onClick），确保即时响应
- 禁止双击缩放：按钮和网格格子都设置 `touch-action: manipulation`
- 质数按钮高度至少 48px，字号至少 17px

## 怪物系统

### 质数池
`[2, 3, 5, 7, 11, 13]`

### 怪物类型

| 类型 | 因数个数 | 生成方式 | 分值 | 视觉 |
|------|---------|---------|------|------|
| 小怪 | 2 | 从质数池随机取2个相乘（允许重复） | 100 | 绿色边框，较小 |
| 大怪 | 3 | 随机取3个相乘 | 200 | 橙色边框，显示"大怪"标签 |
| Boss | 5 | 随机取5个相乘 | 500 | 红色边框，发光效果，显示"BOSS"标签 |

### 怪物配色
```
小怪: 背景 #064e3b, 边框 #10b981, 文字 #6ee7b7
大怪: 背景 #7c2d12, 边框 #f97316, 文字 #fdba74
Boss: 背景 #7f1d1d, 边框 #ef4444, 文字 #fca5a5
```

### 怪物数值显示
- 怪物格子上大字号显示当前数值
- 被命中后数值立即更新（除以射中的质数）
- 数值变为1时怪物消灭，有消失动画
- 根据数值位数自适应字号：≤2位 17px，3位 13px，4位 11px，≥5位 9px

### 下落速度
| 类型 | 速度 |
|------|------|
| 小怪 | 每 **3秒** 下落一格 |
| 大怪 | 每 **2.5秒** 下落一格 |
| Boss | 每 **3秒** 下落一格 |

### 生成规则
- 每 **3秒** 自动生成一只小怪（在随机列的第0行出现）
- 每累计击杀 **3只** 怪物后，额外生成一只**大怪**
- 每累计击杀 **10只** 怪物后，额外生成一只 **Boss**（无特殊技能，模式A中Boss就是大号怪物）

## 射击机制

### 命中判定
1. 玩家点击质数按钮 P
2. 查找当前列（大炮所在列）中**行数最大**（最低位置）的怪物 M
3. 如果 M.value 能被 P 整除：
   - M.value = M.value / P
   - 显示命中闪光（黄色高亮 300ms）
   - 如果 M.value === 1，消灭怪物，加分
4. 如果 M.value 不能被 P 整除：
   - **射错惩罚**（见下方）

### 射错惩罚（全面封锁）
- 所有质数按钮被禁用 **1秒**
- 网格点击也被禁用（不能移动大炮）**1秒**
- 按钮变灰色
- 显示锁定图标和倒计时："🔒 0.8s"
- 屏幕背景闪红色提示
- 连击计数归零
- 得分倍率重置为 1.0×

---

## 计分系统

### 得分
- 消灭小怪：**+100分**（受倍率影响）
- 消灭大怪：**+200分**（受倍率影响）
- 消灭Boss：**+500分**（受倍率影响）

### 扣分
- 小怪到达底部：**-100分**（不受倍率影响）
- 大怪到达底部：**-200分**（不受倍率影响）
- Boss到达底部：**-500分**（不受倍率影响）
- 怪物到底后消失，**不会淘汰玩家**
- 分数可以为负数

### 连击奖励系统
- 系统记录连续正确射击次数（每次命中因数 +1）
- 连续正确 **10次** 后，激活 **1.5× 得分倍率**
- 倍率激活后持续生效，直到射错一次
- 射错后：连击计数归零，倍率回到 1.0×
- 屏幕上显示当前连击数（格式："🔥 7/10"），激活后显示 "×1.5 COMBO!" 闪烁动画
- 倍率只影响**消灭怪物的得分**，不影响到底扣分

## 游戏流程

### 开始界面
1. 显示游戏名称"数字大炮"
2. 简明操作说明（点击列移动 → 按质数射击 → 射错锁定 → 连击加分）
3. 三种怪物的预览卡片
4. "开始对战"按钮

### 游戏中
1. 点击"开始对战"后3-2-1倒计时开始
2. 60秒倒计时，最后10秒倒计时变红
3. 两侧各自独立运行，不互相影响
4. 怪物按规则生成和下落

### 结束界面
1. 显示"⏱ 时间到！"
2. 两侧分数卡片并排显示
3. 每张卡片显示：分数、击杀数、最高连击数
4. 高分方标金色边框 + "🏆 玩家X胜利！"
5. "再来一局"按钮

## Tournament Hub 集成（postMessage 协议）

游戏需要支持通过 URL 参数和 postMessage 与 Tournament Hub 通信。

### URL 参数
游戏应读取以下 URL 参数：
- `playerA`：左侧玩家名称（默认"玩家 A"）
- `playerB`：右侧玩家名称（默认"玩家 B"）
- `mode`：游戏模式（目前只有 "a"）
- `duration`：游戏时长秒数（默认 60）

### 游戏结束时发送消息
游戏结束后，向父窗口发送以下消息：

```javascript
window.parent.postMessage({
  type: "gameEnd",
  scoreA: 850,          // number, 玩家A分数（可为负数）
  scoreB: 620,          // number, 玩家B分数（可为负数）
  winner: "A",          // "A" | "B" | "draw"
  extra: {
    killsA: 12,
    killsB: 8,
    maxComboA: 15,
    maxComboB: 7,
    missedA: 2,         // 漏掉的怪物数
    missedB: 4,
    locksA: 3,          // 被锁定次数
    locksB: 5,
  }
}, "*");
```

### 独立运行
如果不在 iframe 中（直接访问 URL），游戏应正常运行，postMessage 只是额外发送，不影响功能。

## 游戏循环实现要点

### 定时器架构
- 使用 `setInterval` 或 `requestAnimationFrame` 实现游戏主循环
- Tick 间隔约 **100ms**
- 每个 tick 更新：
  - 倒计时
  - 怪物下落累加器
  - 锁定倒计时
  - 命中闪光倒计时

### 两侧独立状态
每侧玩家的状态完全独立，包括：
```javascript
{
  score: 0,
  monsters: [],
  cannon: 2,          // 当前大炮所在列（0-4）
  combo: 0,
  maxCombo: 0,
  mult: 1,            // 得分倍率
  locked: false,
  lockAcc: 0,         // 锁定已经过的时间
  kills: 0,
  spawnAcc: 0,        // 生成怪物累加器
  nextBigAt: 3,       // 下一次出大怪的击杀数
  nextBossAt: 10,     // 下一次出Boss的击杀数
  missed: 0,          // 漏掉的怪物数
  locks: 0,           // 被锁定次数
}
```

### 怪物数据结构
```javascript
{
  id: 唯一ID,
  type: "small" | "big" | "boss",
  col: 0-4,           // 所在列
  row: 0-9,           // 所在行（0=顶部）
  value: 当前数值,
  factors: [原始质因数数组],
  pts: 100|200|500,
  fallMs: 下落间隔毫秒,
  fallAcc: 0,         // 下落累加器
  hitFlash: 0,        // 命中闪光剩余时间
}
```

## 注意事项

1. **触屏优先**：这是在大屏触摸屏上使用的，不需要键盘操作
2. **两侧完全独立**：左右两侧的怪物、分数、连击、锁定互不干扰
3. **不用 localStorage**：不需要保存任何数据
4. **无需音效**：原型阶段不需要
5. **怪物可重叠**：同一格子如果有多个怪物，显示最低（行号最大）的那个，但实际上每个格子应该只有一个怪物。如果新生成的怪物目标列的第0行已有怪物，选择其他空列。
6. **性能**：60秒内屏幕上最多同时存在约20个怪物，不需要特别的性能优化
