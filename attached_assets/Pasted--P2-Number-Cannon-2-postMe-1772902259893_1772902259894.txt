# P2 视觉打磨 — 第二批：炮弹飞行动画 + 怪物造型

## 目标

为 Number Cannon 添加 2 项视觉改进。**不改变任何游戏逻辑、计分规则、命中判定时机或 postMessage 协议。**

---

## 改动 1：炮弹飞行动画

### 现状
玩家按下质数按钮后，命中/射错的判定和效果立即发生，没有任何视觉过渡。

### 目标
按下质数按钮后，一颗带数字的炮弹从大炮位置向上飞向目标怪物（或飞向列顶部表示射空/射错），作为纯视觉装饰。

### 核心原则
- **命中判定时机不变**——射击逻辑仍然在 `useGameLoop.js` 的 `shoot()` 中瞬间完成
- 炮弹是纯视觉层的叠加动画，不影响游戏状态
- 飞行时间很短（~200ms），保持快节奏手感

### 实现方案

**新增组件 `Projectile.jsx`**：
- 渲染一个圆形弹丸，内部显示质数数字
- 用 CSS animation 从起点飞到终点，动画结束后自动移除

**在 `PlayerSide.jsx` 中管理弹丸状态**：
- 维护一个 `projectiles` 数组（React state）
- 每次射击时（onShoot 回调触发后），添加一个弹丸对象：
  ```js
  {
    id: Date.now() + Math.random(),
    prime: 射出的质数,
    startX: 大炮列的中心 x 坐标,
    startY: 大炮行的 y 坐标（网格底部）,
    endX: 目标怪物的中心 x 坐标（如果该列有怪物）或同列顶部,
    endY: 目标怪物的 y 坐标 或 网格顶部,
    hit: 是否命中（用于颜色区分）,
  }
  ```
- 弹丸动画结束后（200ms），从数组移除

**如何获取坐标**：
- 需要知道网格容器的实际像素位置和尺寸
- 通过 ref 获取网格容器的 `getBoundingClientRect()`
- 起点 = 列中心 x, 网格底部 y
- 终点 = 列中心 x, 目标怪物的行对应 y（如果列中有怪物）；如果列为空则飞到网格顶部

**如何知道目标怪物位置**：
- 射击时遍历 `player.monsters`，找到当前大炮列中 `row` 最大的怪物
- 如果该列没有怪物，终点设为网格顶部

**弹丸视觉设计**：
- 圆形，直径约 28px
- 命中时：金黄色背景 `#fbbf24`，黑色数字
- 射错时：红色背景 `#ef4444`，白色数字
- 射空（列无怪物不会发生射击，但以防万一）：灰色
- 带轻微拖尾效果：`box-shadow: 0 4px 12px rgba(251, 191, 36, 0.6)`

**CSS 动画**：
```css
@keyframes projectileFly {
  0% {
    opacity: 1;
  }
  80% {
    opacity: 1;
  }
  100% {
    opacity: 0;
  }
}
```
- 弹丸使用绝对定位，position 从 `(startX, startY)` 到 `(endX, endY)` 的 transition（200ms ease-out）
- 实现方式：初始渲染在 startY，下一帧（requestAnimationFrame 或 setTimeout 0）设置目标位置触发 transition

**整合流程**：
1. 玩家按下质数按钮
2. `onShoot(side, prime)` 被调用 → `useGameLoop` 中立即完成命中判定
3. 在 `PlayerSide` 的 shoot wrapper 中，计算弹丸坐标并添加到 projectiles 数组
4. 弹丸渲染并飞行 200ms
5. 飞行结束后移除弹丸

### 注意事项
- 弹丸是叠加在网格上方的绝对定位元素（和怪物同一层级或更高 z-index）
- 快速连射时可能同时有多颗弹丸在飞，用 id 区分
- 弹丸不需要碰撞检测——纯装饰
- 左右两侧各自独立管理自己的弹丸

## 改动 2：怪物造型区分

### 现状
三种怪物（小怪/大怪/Boss）只靠背景色和边框色区分（绿/橙/红），都是相同的圆角矩形+数字。大怪有"大怪"文字标签，Boss 有"BOSS"标签和 box-shadow。

### 目标
三种怪物有明显不同的视觉形态，在课堂大屏上一眼可辨。使用 CSS 造型 + emoji，不引入图片资源。

### 设计方案

**小怪 — 圆形小虫**：
- 形状：圆形（`border-radius: 50%`）
- 顶部有两个小触角（用 `::before` 和 `::after` 伪元素，或两个小圆点 div）
- 保持现有绿色系配色（`#064e3b` 背景，`#10b981` 边框）
- 内部显示数字（去掉文字标签，因为小怪没有标签）
- 可以加两个小眼睛（用两个白色小圆点）在数字上方

**大怪 — 带角的方形怪物**：
- 形状：略大的圆角方形，顶部有两个三角形"角"
- 三角形用 CSS border trick 或 `clip-path` 实现
- 保持橙色系配色（`#7c2d12` 背景，`#f97316` 边框）
- 去掉"大怪"文字标签，靠造型区分
- 可以有一个锯齿形的嘴巴（用底部 border 变化暗示）

**Boss — 大型带皇冠的圆角方形**：
- 形状：比大怪更大（如果格子允许就占满格子），圆角方形
- 顶部有三个尖刺形成皇冠效果（CSS `clip-path` 或三个小三角）
- 保持红色系配色（`#7f1d1d` 背景，`#ef4444` 边框）
- 脉动发光效果：`box-shadow` 用 CSS animation 做呼吸灯
- "BOSS" 标签保留，放在数字下方，字号更小
- 可以有两个发光的红色眼睛

### 实现要点

- 所有造型改动都在 `MonsterCell.jsx` 中完成
- 用条件渲染根据 `monster.type` 切换不同的 JSX 结构
- 触角、角、皇冠等装饰元素用绝对定位，不影响网格布局
- 数字仍然居中显示，字号逻辑不变
- `dying` 动画仍然正常工作（scale + fade 应用到整个怪物容器）
- `hitFlash` 仍然正常工作（改变背景色为金黄色）

### CSS 补充（加到 App.css）
```css
@keyframes bossPulse {
  0%, 100% { box-shadow: 0 0 8px rgba(239, 68, 68, 0.4); }
  50% { box-shadow: 0 0 20px rgba(239, 68, 68, 0.8); }
}
```
